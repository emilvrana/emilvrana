The Feedback Loop Tax

July 17, 2026

Every AI system generates feedback loops. Most of them are invisible, and most of them are expensive.

A feedback loop in an AI system is any cycle where the output of the system becomes part of its input. Some are intentional — reinforcement learning, RLHF, evaluation pipelines. Others are accidental — model outputs polluting training data, agents reading their own logs, users adapting to model quirks in ways that reinforce those quirks.

The intentional loops are expensive by design. The accidental loops are expensive by surprise. And both of them compound.

The Tax You Don't See

Here's a pattern I see repeatedly:

  1. Deploy an agent. It works well on the first 100 tasks.
  2. The agent generates outputs — summaries, classifications, decisions — that get stored somewhere.
  3. Those outputs become context for future runs. The agent starts seeing its own prior work.
  4. Subtle biases in the initial outputs get reinforced. The agent converges on a narrower and narrower view of what "correct" looks like.
  5. Nobody notices because the degradation is gradual and the agent still produces plausible-looking output.

This is the feedback loop tax: the invisible cost of a system gradually eating its own tail.

It shows up in three places.

Model output as training data

You fine-tune a model on user interactions. The model gets better at what it already does well, and worse at edge cases it avoided. The next fine-tuning cycle amplifies this. After three cycles, you've built a model that's very confident and very narrow.

Agent context as prior

An agent reads its own past decisions to inform new ones. "Last time I classified this as X, so X is probably right again." This isn't reasoning — it's anchoring. The agent isn't learning from outcomes. It's learning from its own past guesses, treating them as ground truth.

User behavior as signal

Users adapt to your model's strengths. If the model is good at Python but weak at Rust, users stop asking it about Rust. Your evaluation metrics improve — not because the model got better at Rust, but because nobody asks anymore. The metrics lie.

The Math of Loop Decay

Feedback loops in AI systems don't cause catastrophic failure. They cause something worse: slow, invisible degradation.

Imagine your agent has a 5% error rate on a particular task type. Each time it processes that task type, there's a 5% chance it produces a wrong answer that gets stored and referenced later. That wrong answer becomes part of the context for future decisions.

After N cycles, the error rate isn't 5%. It's closer to 5% × (1 + decay factor). The decay factor depends on how often the agent references its own past outputs, how much weight it gives them, and how quickly wrong answers propagate through the system.

In practice, I've seen systems with 5% initial error rates degrade to 15–20% effective error rates over 3–6 months — entirely through feedback loops, with no change in the underlying model.

Breaking the Loop

The solution isn't to avoid feedback loops entirely. That's impossible in any system that learns. The solution is to make the loops visible and to inject correction signals.

Separate generation from validation

Never let the same system that produces an output be the sole judge of that output's quality. Build independent evaluation. This can be a simpler model, a rule-based system, or human review — but it has to be outside the loop.

Audit what your agent reads

Most agent systems I see have a context window that grows organically. The agent pulls in conversation history, tool results, prior decisions, documents, logs. Nobody audits what's in there. Start auditing. If your agent is reading its own past outputs without an explicit reason, you have an accidental feedback loop.

Inject noise on purpose

Counterintuitive but effective: deliberately introduce variation into your system. Don't always rank results by the same metric. Don't always select the top-k most similar documents. Don't always use the same model for the same task. A small amount of controlled randomness prevents convergence.

Measure against held-out ground truth

Keep a set of tasks with known correct answers. Run them regularly. Don't just check if the system produces plausible output — check if it produces correct output compared to a standard that doesn't change.

Quarantine self-referential data

If your agent's past decisions are stored as context, tag them as such. Then weight them lower than external, verified data. "The agent previously said X" should carry less weight than "the documentation says Y."

The Meta-Pattern

The feedback loop tax is a special case of a broader principle: any system that optimizes against its own output without external correction will converge. This is true for language models, recommendation engines, ranking algorithms, and economic markets.

In machine learning, we call this distribution shift. In economics, we call it a bubble. In AI systems, we call it "the model seems fine" — until it isn't.

The fix is always the same: external grounding. A system needs input from outside its own output loop. Not constantly, but regularly enough to prevent drift.

Build your AI systems with escape hatches — independent evaluation, external benchmarks, human review checkpoints, and data freshness policies. The loops will form whether you plan for them or not. The question is whether you've built in the mechanisms to break them before the tax compounds.
